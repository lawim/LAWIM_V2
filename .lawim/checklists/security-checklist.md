# Security Checklist

Utiliser cette checklist avant toute validation de securite ou de mise en production.

## Acces
- [ ] Les roles et permissions sont definis.
- [ ] Le moindre privilege est applique.
- [ ] Les acces sensibles sont traces.
- [ ] Les comptes ou secrets partages sont evites.

## Donnees
- [ ] Aucun secret en clair n'est present.
- [ ] Les donnees sensibles ne sont pas exposees dans les logs.
- [ ] Les flux de donnees critiques sont identifies.
- [ ] Les risques de fuite sont traites.

## Surface d'attaque
- [ ] Les dependances sensibles sont connues.
- [ ] Les points d'entree sont documentes.
- [ ] Les risques majeurs sont listes.
- [ ] Les contre-mesures sont explicites.

## Validation
- [ ] Les resultats de revue securite sont traces.
- [ ] Les ecarts ont un plan de mitigation.
- [ ] La decision finale est reliee au ticket.
