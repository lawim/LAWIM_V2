# Documentation Checklist

Utiliser cette checklist avant toute publication ou mise a jour documentaire.

## Coherence
- [ ] Le document correspond au bon perimetre.
- [ ] Le titre et l'objectif sont explicites.
- [ ] Les references sont correctes.
- [ ] Les doublons sont evites.

## Traite
- [ ] La date et la version sont presentes si necessaires.
- [ ] Les decisions ou validations liees sont tracees.
- [ ] Le document est aligne avec le PCC et le workflow officiel.

## Lisibilite
- [ ] Le vocabulaire est stable.
- [ ] Les sections sont faciles a parcourir.
- [ ] Les listes ou tables sont coherentes.

## Gouvernance
- [ ] Aucune regle fondatrice n'est redefinie.
- [ ] Aucune Constitution ou referentiel metier n'est modifie.
- [ ] Le proprietaire du document est identifiable.
