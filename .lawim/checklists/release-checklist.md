# Release Checklist

Utiliser cette checklist avant toute mise en release.

## Preparation
- [ ] La version est fixee.
- [ ] Le scope de release est documente.
- [ ] Les tickets inclus sont listes.
- [ ] Le changelog est pret.

## Qualite
- [ ] Les validations architecture, QA, security et integration sont completes.
- [ ] Les points ouverts sont explicites.
- [ ] Les risques de release sont connus.

## Branche et livrable
- [ ] La branche de release est identifiee.
- [ ] Les corrections de derniere minute sont tracees.
- [ ] Le tag ou la version cible est confirme.

## Rollback
- [ ] Un plan de retour arriere existe.
- [ ] Les prerequis de restauration sont connus.
- [ ] La decision de mise en production est tracable.
