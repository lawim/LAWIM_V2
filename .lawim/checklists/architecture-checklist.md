# Architecture Checklist

Utiliser cette checklist avant toute validation technique majeure.

## Perimetre
- [ ] Le perimetre du ticket est explicite.
- [ ] Aucune modification de Constitution ou de referentiel metier n'est proposee.
- [ ] Le ticket reste limite a un besoin principal.

## Conception
- [ ] Les couches et frontieres sont respectees.
- [ ] Les dependances sont explicites.
- [ ] Les conventions techniques sont coherentes.
- [ ] Les contrats entre composants sont documentes.

## Qualite
- [ ] Les risques de dette technique sont identifies.
- [ ] Les points de fragilite sont notes.
- [ ] Les impacts transverses sont enumeres.
- [ ] Les choix sont reversibles si necessaire.

## Traceabilite
- [ ] La decision est reliee a un ticket.
- [ ] Les reservations sont consignees.
- [ ] Les references utiles sont ajoutees.
