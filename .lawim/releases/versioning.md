# Versioning

## Regle
LAWIM_V2 utilise un versioning semantique au format MAJOR.MINOR.PATCH.

## Signification
- MAJOR: changement incompatible de gouvernance, d'architecture ou de contrat majeur;
- MINOR: ajout non cassant, evolution de workflow ou extension documentaire;
- PATCH: correction, clarification ou ajustement sans changement de perimetre.

## Regles de release
- chaque version doit etre reliee a un scope documente;
- chaque release doit citer les tickets inclus;
- aucune version ne doit etre publiee sans trace de validation;
- les corrections de release ne modifient pas le sens du referentiel fondateur.

## Convention de tag
- vMAJOR.MINOR.PATCH
