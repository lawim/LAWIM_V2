# Operating Model

## Principle
LAWIM_V2 est pilote par une chaine de decision centree sur le Directeur General d'Implementation.

## Actors
- Directeur General: decide, arbitre, valide.
- PMO: suit, consolide, rapporte.
- Agents specialises: executent uniquement les tickets transmis.
- QA: verifie la conformite aux criteres d'acceptation.

## Flow
1. Le Directeur General ouvre le sprint.
2. Le Directeur General ouvre les tickets.
3. Le PMO met a jour le PCC et le suivi.
4. L'agent specialise execute le ticket.
5. Le QA verifie les livrables.
6. Le Directeur General accepte ou refuse.
7. Le PCC est mis a jour.

## Constraints
- Aucun agent specialise ne recherche le backlog.
- Aucun agent specialise ne reconstruit les sprints.
- Toute decision de gouvernance revient au Directeur General.
- Toute trace doit rester dans le perimetre du depot.
