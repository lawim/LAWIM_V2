# Mission 13.2 - Lot 6 - Cockpit Disaster Recovery Tab

## Objective

Expose the DRF status, recovery bundles, validation evidence, version state,
and bundle download controls in the admin Cockpit without exposing secrets.

## Delivered Files

- `code/lawim_v2/backup/recovery.py`
- `code/lawim_v2/server.py`
- `frontend/packages/api-sdk/src/index.ts`
- `frontend/apps/admin/src/DisasterRecoveryPage.tsx`
- `frontend/apps/admin/src/App.tsx`
- `frontend/tests/disaster-recovery.test.tsx`
- `tests/test_disaster_recovery_validation.py`
- `frontend/dist/`

## Implemented Behavior

The DRF cockpit now provides:

- a dedicated "Disaster Recovery" route in the admin navigation;
- recovery status cards for bundle root, latest bundle, validation state,
  Git state, versions, and checklist evidence;
- bundle inventory with download actions for the latest bundle and each listed
  bundle;
- explicit validation checks surfaced from the backend DRF validator;
- a recovery score preview based on bundle freshness and validation signals;
- bundle archive downloads through the DRF API;
- contract types and SDK helpers for DRF status, bundle listing, validation,
  and archive download.

The backend now supports:

- `GET /api/v2/backup/recovery`
- `GET /api/v2/backup/recovery/bundles`
- `GET /api/v2/backup/recovery/latest`
- `GET /api/v2/backup/recovery/validation`
- `POST /api/v2/backup/recovery/validate`
- `GET /api/v2/backup/recovery/bundles/{bundle_id}/download`
