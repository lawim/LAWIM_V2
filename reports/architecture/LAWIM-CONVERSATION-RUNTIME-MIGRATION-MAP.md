## services.py Wires ConversationStateEngine into CommunicationService

At `services.py:193-219`:

```python
from .conversation.state.engine import ConversationStateEngine

from .conversation.state.resolver import ConversationResolver

_conv_repo = ConversationStateRepository(_db)
_conv_resolver = ConversationResolver()
_conv_engine = ConversationStateEngine(_conv_repo, _conv_resolver)
```

This engine instance is passed to `CommunicationService.__init__()` as `conversation_state_engine`, making it available to `_generate_ai_reply()`.

## Validator Enhanced

The `ConversationResponseValidator` (at `conversation/state/validator.py`) was enhanced to detect:

| Category | Patterns | Status |
|----------|----------|--------|
| Neutral assistant | "neutral assistant", "I cannot make business decisions", "provide more context for your request" | `REPAIR` |
| External referrals | "Jumia", "SeLoger", "Leboncoin", "Facebook", "Lamudi" | `REPAIR` / `BLOCK` |
| Translation | "french for", "in english", "français signifie" | `REPAIR` |
| Grammar correction | "correct spelling is", "the correct phrasing", "you wrote" | `REPAIR` |
| Multiple questions | `response.count("?") > plan.maximum_questions` | `REPAIR` (replaced with plan.next_question_text) |
