# DevOps Rules

- Toute action DevOps doit rester dans le perimetre infrastructure.
- Docker, Docker Compose, Nginx, OVH, CI/CD, logs et monitoring sont le coeur du perimetre.
- Aucun changement DevOps ne doit introduire de logique metier.
- Les environnements doivent etre definis, coherents et documentes.
- Les secrets doivent etre geres hors du code source.
- Les deploiements doivent etre reproductibles et traces.
