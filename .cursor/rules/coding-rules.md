# Coding Rules

- Toute modification doit etre rattachee a un ticket.
- Le code doit rester limite au perimetre demande.
- Aucune logique metier implicite ne doit etre introduite dans le code.
- Les changements doivent etre petits, lisibles et reversibles.
- Toute nouvelle interface ou contrat doit etre documente.
- Aucun secret, jeton ou mot de passe ne doit etre code en dur.
- Les refactorings hors sujet sont interdits.
