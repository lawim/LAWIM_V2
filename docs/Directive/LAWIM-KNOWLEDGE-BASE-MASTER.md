# Architecture maîtresse de la base de connaissances

Version 1.0

Date : 2026-06-27

---

# 1. Objectif

Ce document définit la structure de la future base de connaissances LAWIM.

Cette base doit alimenter directement LAWIM AI, le matching, le support, le reporting et la documentation.

---

# 2. Principes

* une connaissance doit être identifiable ;
* une connaissance doit être versionnée ;
* une connaissance doit pouvoir être validée ;
* une connaissance doit pouvoir être reliée à une source ;
* une connaissance doit pouvoir être multilingue ;
* une connaissance ne doit pas être appliquée automatiquement sans validation humaine lorsque le sujet est sensible.

---
