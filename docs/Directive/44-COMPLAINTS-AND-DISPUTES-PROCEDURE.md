# Procédure de traitement des réclamations et litiges

Version 1.0

---

# 1. Objectif

Le présent document définit la gestion des réclamations, litiges et incidents relationnels LAWIM.

---

# 2. Périmètre

Cette procédure couvre :

* réclamation utilisateur ;
* litige entre utilisateurs ;
* litige avec une agence ;
* litige avec un partenaire ;
* litige lié à un paiement ;
* litige lié à une visite ;
* litige lié à un bien ;
* litige lié à une information fausse.

---

# 3. Niveaux

* simple demande ;
* réclamation ;
* litige ;
* litige sensible ;
* fraude suspectée ;
* urgence.

---

# 4. Acteurs concernés

* support ;
* opérations ;
* conformité ;
* sécurité ;
* finance ;
* partenariat ;
* direction si nécessaire.

---

# 5. Étapes officielles

1. Réception.
2. Classification.
3. Priorisation.
4. Accusé de réception.
5. Collecte des éléments.
6. Analyse.
7. Médiation éventuelle.
8. Décision.
9. Communication.
10. Clôture.
11. Archivage.
12. Reporting.

---

# 6. Délais internes de traitement

Objectifs de traitement :

* simple demande : accusé de réception le jour même ;
* réclamation : première réponse sous 24 heures ouvrées ;
* litige : première analyse sous 48 heures ouvrées ;
* litige sensible : escalade immédiate ;
* fraude suspectée : gel conservatoire sans attendre ;
* urgence : prise en charge prioritaire.

---

# 7. Collecte des éléments

Conserver :

* messages ;
* captures ;
* références de paiement ;
* identifiants ;
* dates ;
* liens ;
* preuves de visite ;
* tout document utile.

---

# 8. Confidentialité

La confidentialité doit être respectée à chaque étape.

Il est interdit :

* d'exposer un dossier sensible à un tiers non autorisé ;
* de partager une adresse non nécessaire ;
* de publier des preuves hors du cadre autorisé ;
* de masquer un risque grave.

---

# 9. Escalade

Escalader vers :

* sécurité si fraude ou abus ;
* finance si paiement concerné ;
* partenariat si partenaire impliqué ;
* direction si impact réputationnel ou juridique ;
* support si demande de clarification.

---

# 10. Notifications

Notifier :

* l'auteur de la réclamation ;
* les parties concernées ;
* le responsable interne ;
* la sécurité ou la finance si nécessaire ;
* la direction si la décision est sensible.

---

# 11. Reporting

Suivre au minimum :

* nombre de réclamations reçues ;
* nombre de litiges ouverts ;
* délais de réponse ;
* délais de clôture ;
* taux de résolution ;
* motifs récurrents ;
* cas liés aux paiements ;
* cas liés aux biens ;
* cas liés aux partenaires.

---

# 12. Règles absolues

* ne jamais promettre une issue avant analyse ;
* ne jamais supprimer une preuve ;
* ne jamais bloquer un dossier sans motif ;
* ne jamais contourner la validation humaine ;
* ne jamais créer de commission immobilière ;
* ne jamais clore un litige sensible sans justification.

---

# 13. Objectif final

Cette procédure garantit un traitement juste, traçable et cohérent des réclamations et litiges LAWIM.
