# Procédures opérationnelles officielles

Version 1.0

---

# 1. Objectif

Le présent document définit les procédures opérationnelles de LAWIM.

Il sert au support, à l'exploitation, à la coordination interne, à l'audit et à l'escalade.

Il ne modifie aucun référentiel technique.

---

# 2. Périmètre

Les procédures opérationnelles couvrent notamment :

* l'accueil des partenaires ;
* la vérification des biens ;
* les réclamations et litiges ;
* la certification des agences ;
* la fraude ;
* la suspension des partenaires ;
* le suivi commercial ;
* le reporting opérationnel ;
* l'archivage des décisions.

---

# 3. Différence entre référentiels et procédures

Les référentiels techniques définissent les règles officielles du système LAWIM.

Les procédures opérationnelles définissent la manière d'agir au quotidien.

Les procédures :

* ne créent pas de moteur ;
* ne modifient pas les workflows techniques ;
* ne changent pas le modèle économique ;
* ne remplacent pas la validation humaine ;
* s'appuient sur les référentiels existants.

---

# 4. Acteurs concernés

Les acteurs opérationnels sont notamment :

* exploitation ;
* support ;
* partenariat ;
* certification ;
* conformité ;
* sécurité ;
* finance ;
* direction ;
* équipe commerciale ;
* équipe terrain.

---

# 5. Niveaux d'urgence

LAWIM utilise quatre niveaux d'urgence.

* P1 critique : fraude, incident de sécurité, litige majeur, paiement bloqué, fuite de données.
* P2 élevé : partenaire sensible, bien suspect, réclamation récurrente, désaccord important.
* P3 normal : onboarding, vérification standard, suivi partenaire, contrôle de routine.
* P4 information : rappel, reporting, archivage, suivi non bloquant.

Chaque niveau impose un délai de traitement interne adapté.

---

# 6. Principes opérationnels

* toute action doit être tracée ;
* toute décision sensible doit être validée par un humain ;
* toute suspicion doit être documentée ;
* toute communication doit rester cohérente avec les référentiels LAWIM ;
* aucune commission immobilière ne doit être annoncée ;
* les services payants doivent être présentés clairement et séparément.

---

# 7. Chaîne opérationnelle commune

1. Détection ou demande.
2. Qualification du sujet.
3. Attribution à un responsable.
4. Collecte des preuves ou éléments utiles.
5. Traitement.
6. Validation humaine.
7. Notification.
8. Archivage.
9. Reporting.

---

# 8. Responsabilités internes

* le support reçoit et qualifie ;
* l'exploitation traite et coordonne ;
* la conformité valide les cas sensibles ;
* la sécurité gère les cas de fraude ou de risque ;
* la direction arbitre les cas exceptionnels ;
* l'équipe commerciale exécute les procédures liées aux partenaires et aux ventes.

# 10. Reporting

Les procédures opérationnelles doivent produire au minimum :

* un journal quotidien ;
* un suivi hebdomadaire ;
* un récapitulatif mensuel ;
* une liste des dossiers ouverts ;
* une liste des décisions prises ;
* une liste des cas escaladés.

---

# 11. Archivage et audit

Chaque dossier opérationnel doit conserver :

* la date de réception ;
* le responsable ;
* les preuves collectées ;
* la décision ;
* le motif ;
* les notifications envoyées ;
* la date de clôture ;
* la référence d'archivage.

L'archivage sert l'audit, le contrôle interne et le suivi historique.

---

# 12. Règles absolues

* aucune procédure ne doit contredire la Constitution ;
* aucune procédure ne doit créer une commission sur transaction immobilière ;
* aucune procédure ne doit supprimer une validation humaine ;
* aucune procédure ne doit créer un moteur ;
* aucune procédure ne doit réécrire les référentiels techniques ;
* aucune procédure ne doit masquer une information critique.

---

# 13. Objectif final

Les procédures opérationnelles garantissent que LAWIM peut être exploité de manière claire, traçable, rapide et conforme, sans modifier son architecture validée.
