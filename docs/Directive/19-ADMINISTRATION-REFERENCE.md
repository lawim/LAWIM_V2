# Référentiel officiel de l'administration

Version 1.0

---

# CHAPITRE 1 — OBJECTIF

Le présent document définit les règles officielles d'administration de LAWIM.

Il constitue la référence unique pour :

* la supervision ;
* la gestion des utilisateurs ;
* la gestion des rôles ;
* la gestion des agences ;
* la gestion des partenaires ;
* la validation documentaire ;
* la maintenance ;
* le support ;
* la supervision des paiements Campay.

---

# CHAPITRE 2 — PRINCIPE FONDAMENTAL

L'administration ne remplace aucun moteur propriétaire.

Elle supervise, valide et coordonne les opérations sensibles.

Toute action administrative doit être tracée et, lorsque nécessaire, validée par un second niveau d'autorité.

---

# CHAPITRE 3 — DOMAINES D'ACTION

L'Administration Engine couvre notamment :

* les comptes ;
* les rôles ;
* les organisations ;
* les validations ;
* les paramètres système ;
* les incidents ;
* les exports ;
* la supervision des flux sensibles.

---

# CHAPITRE 4 — VALIDATION DOCUMENTAIRE

Les documents sensibles peuvent être contrôlés par l'administration selon les règles du référentiel de stockage et du référentiel de sécurité.

L'administration peut :

* demander une correction ;
* refuser une pièce ;
* déclencher une revue humaine ;
* archiver un dossier conforme.

---

# CHAPITRE 5 — PAIEMENTS CAMPAY

L'administration doit pouvoir :

* consulter les transactions ;
* filtrer les statuts ;
* vérifier les confirmations ;
* suivre les échecs ;
* contrôler le rapprochement ;
* lancer un support ou un remboursement si applicable.

L'administration ne peut jamais confirmer un paiement sans preuve serveur valide.

---

# CHAPITRE 6 — INCIDENTS ET SUPERVISION

Les incidents à suivre comprennent notamment :

* indisponibilité OVH ;
* incident base de données ;
* incident Google Drive ;
* incident stockage local ;
* incident sécurité ;
* incident Campay ;
* incident webhook.

Chaque incident doit être tracé et escaladé selon la criticité.

---

# CHAPITRE 7 — INTERACTIONS

L'Administration Engine coopère notamment avec :

* Role Engine ;
* Security Engine ;
* Storage Lifecycle Manager ;
* Reporting Engine ;
* Dashboard Engine ;
* Campay Payment Engine ;
* Continuous Learning Engine.

---

# CHAPITRE 8 — TESTS

Les tests administratifs doivent vérifier :

* création d'utilisateur ;
* changement de rôle ;
* validation documentaire ;
* suivi des incidents ;
* visibilité des paiements ;
* export des données ;
* contrôle des accès.

---

# CHAPITRE 10 — TRACKING MARKETING ET SUPERVISION

L'administration doit pouvoir superviser :

* les campagnes ;
* les publications ;
* les codes de tracking ;
* les redirections ;
* les sources de leads ;
* les attributions de conversion ;
* les exportations marketing ;
* les anomalies de déduplication ;
* les cohortes par canal, acteur, ville, langue et type de bien.

Cette supervision enrichit les référentiels existants sans créer de dashboard supplémentaire.

---

# CHAPITRE 11 — OBJECTIF FINAL

Le référentiel d'administration garantit une supervision fiable, tracée et gouvernée de LAWIM.
