# Référentiel officiel de LAWIM AI

Version 1.0

---

# CHAPITRE 1 — OBJECTIF

Le présent document définit les règles officielles de LAWIM AI.

Il constitue la référence unique pour :

* l'assistance conversationnelle ;
* le résumé automatique ;
* les recommandations ;
* le matching assisté ;
* l'analyse documentaire ;
* la mémoire contextuelle ;
* le Knowledge Graph.

---

# CHAPITRE 2 — MISSION

LAWIM AI assiste l'utilisateur et les équipes LAWIM sans remplacer la décision humaine.

Il peut :

* reformuler ;
* résumer ;
* expliquer ;
* recommander ;
* repérer des incohérences ;
* assister la qualification ;
* aider à la priorisation.

Quand l'utilisateur exprime une intention, LAWIM AI qualifie le besoin de manière progressive: il pose seulement les questions utiles, réutilise les réponses pour préparer la recherche ou le matching, puis propose la prochaine action logique.

---

# CHAPITRE 3 — CAS D'USAGE AUTORISÉS

LAWIM AI peut être utilisé pour :

* l'aide à la recherche ;
* l'extraction d'informations ;
* l'assistance au matching ;
* la qualification conversationnelle progressive des besoins ;
* l'analyse de documents ;
* l'OCR éventuel ;
* la lecture d'images ou de vidéos si la fonctionnalité existe ;
* les synthèses de reporting ;
* la détection automatique de la langue ;
* la réponse dans la langue de l'utilisateur ;
* la traduction des recherches et des réponses ;
* la proposition de synonymes et variantes ;
* l'enrichissement du dictionnaire métier sous validation humaine.

---

# CHAPITRE 4 — MÉMOIRE ET KNOWLEDGE GRAPH

LAWIM AI peut exploiter :

* la mémoire conversationnelle ;
* l'historique des matchings ;
* les retours utilisateurs ;
* les retours partenaires ;
* le Knowledge Graph ;
* les référentiels LAWIM.

La mémoire n'autorise jamais la création automatique d'une règle métier.

---

# CHAPITRE 5 — PROMPT ENGINEERING ET CONTEXTE

Les prompts doivent être :

* explicites ;
* versionnés ;
* traçables ;
* compatibles avec les référentiels ;
* protégés contre les injections dangereuses.

La récupération de contexte doit respecter les permissions.

---

# CHAPITRE 6 — LIMITES ET VALIDATION HUMAINE

LAWIM AI ne peut jamais :

* modifier seul une règle métier ;
* modifier seul un tarif ;
* modifier seul un workflow ;
* valider seul un paiement ;
* contourner un moteur propriétaire.

Toute recommandation critique doit pouvoir être relue et validée par un humain.

---

# CHAPITRE 7 — INTERACTIONS AVEC LES AUTRES MOTEURS

LAWIM AI coopère notamment avec :

* Conversation Engine ;
* Matching Engine ;
* Reporting Engine ;
* Dashboard Engine ;
* Security Engine ;
* Administration Engine ;
* Continuous Learning Engine ;
* Campay Payment Engine.

LAWIM AI ne remplace jamais le moteur source.

---

# CHAPITRE 8 — SÉCURITÉ IA

LAWIM AI doit respecter :

* la confidentialité ;
* les permissions ;
* les restrictions documentaires ;
* l'absence de fuite de secrets ;
* l'absence de génération de commission immobilière fictive.

Les sorties de l'IA doivent être filtrées lorsque nécessaire.

---

# CHAPITRE 9 — TESTS ET AUDIT

Les tests doivent couvrir :

* résumé correct ;
* recommandation cohérente ;
* absence d'hallucination métier critique ;
* respect des permissions ;
* respect du modèle économique ;
* respect des règles de paiement ;
* respect des langues officielles.

---

# CHAPITRE 10 — TRACKING MARKETING ET OPTIMISATION

LAWIM AI peut exploiter les événements de tracking, les campagnes, les publications, les canaux, les acteurs, les conversions et les revenus de services pour :

* identifier les meilleurs canaux ;
* identifier les meilleurs horaires et jours ;
* identifier les meilleures villes, quartiers et types de biens ;
* analyser les titres, textes, appels à l'action et visuels ;
* comparer les langues et variantes de contenu ;
* proposer des optimisations de campagne et de publication ;
* expliquer les performances observées.

LAWIM AI ne peut jamais appliquer automatiquement une optimisation marketing sensible.

Toute recommandation reste soumise à validation humaine.

---

# CHAPITRE 11 — OBJECTIF FINAL

LAWIM AI doit renforcer la qualité, la vitesse et la pertinence de LAWIM sans diminuer la maîtrise humaine.
