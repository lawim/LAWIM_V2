# Procédure de gestion de la fraude

Version 1.0

---

# 1. Objectif

Le présent document définit la détection et le traitement des fraudes dans LAWIM.

---

# 2. Périmètre

Cette procédure couvre notamment :

* faux bien ;
* faux propriétaire ;
* faux agent ;
* faux document ;
* usurpation d'identité ;
* double publication ;
* prix suspect ;
* demande de paiement hors LAWIM ;
* fraude Campay ;
* faux partenaire ;
* manipulation de tracking ;
* faux clics ;
* spam ;
* abus.

---

# 3. Niveaux de risque

* faible ;
* moyen ;
* élevé ;
* critique.

---

# 4. Acteurs concernés

* sécurité ;
* conformité ;
* support ;
* opérations ;
* partenariat ;
* finance ;
* direction si urgence.

---

# 5. Étapes officielles

1. Signalement ou détection.
2. Gel éventuel.
3. Analyse.
4. Collecte des preuves.
5. Décision.
6. Suspension éventuelle.
7. Notification.
8. Clôture.
9. Archivage.
10. Reporting.

---

# 6. Mesures conservatoires

Selon la gravité :

* masquer temporairement un bien ;
* suspendre une publication ;
* bloquer l'activation d'un partenaire ;
* demander des justificatifs ;
* geler une mise en relation ;
* alerter la sécurité ;
* demander une revue humaine immédiate.

---

# 7. Preuves à conserver

* captures d'écran ;
* conversations ;
* références de paiement ;
* pièces justificatives ;
* éléments de tracking ;
* journaux de décision ;
* toute information utile à l'audit.

---

# 8. Lien avec les autres référentiels

Cette procédure doit rester cohérente avec :

* Security Engine ;
* Reporting Engine ;
* Continuous Learning Engine ;
* Campay Payment Engine ;
* les procédures de suspension ;
* les procédures de litiges.

---

# 9. Notifications

Notifier selon le cas :

* l'utilisateur concerné ;
* le partenaire concerné ;
* l'agence concernée ;
* la sécurité ;
* la conformité ;
* la direction si le risque est élevé.

---

# 10. Reporting

Suivre au minimum :

* nombre d'alertes fraude ;
* nombre de cas gelés ;
* nombre de suspensions ;
* motifs les plus fréquents ;
* temps moyen d'analyse ;
* taux de récurrence ;
* cas liés au tracking ;
* cas liés aux paiements ;
* cas liés aux documents.

---

# 11. Règles absolues

* ne jamais inventer une fraude ;
* ne jamais supprimer une preuve ;
* ne jamais agir sans motif documenté ;
* ne jamais appliquer une sanction définitive sans validation humaine, sauf urgence conservatoire ;
* ne jamais créer de commission immobilière ;
* ne jamais laisser une fraude visible sans traitement.

---

# 12. Objectif final

Cette procédure protège LAWIM, ses utilisateurs et ses partenaires contre les abus, la falsification et la manipulation.
