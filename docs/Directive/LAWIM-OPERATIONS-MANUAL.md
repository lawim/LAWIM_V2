# Manuel interne des opérations

Version 1.0

Date : 2026-06-27

---

# 1. Objectif

Ce manuel décrit l'organisation quotidienne de LAWIM.

Il sert aux équipes support, produit, opérations, partenariat, commercial, marketing, RH et direction.

---

# 3. Organisation quotidienne

Chaque journée doit couvrir :

* suivi des tickets ;
* suivi des partenaires ;
* suivi des biens ;
* suivi des réclamations ;
* suivi des paiements ;
* suivi des anomalies ;
* suivi des priorités ;
* suivi des urgences.

---

# 4. Réunions

Types de réunions :

* daily court ;
* point support ;
* point produit ;
* point exploitation ;
* point commercial ;
* point partenariat ;
* point qualité ;
* point direction.

Chaque réunion doit produire un compte rendu court.

---

# 5. KPI

KPI à suivre :

* volume de publications ;
* volume de partenaires ;
* volume de vérifications ;
* volume de réclamations ;
* volume de suspensions ;
* volume de paiements ;
* temps de traitement ;
* taux de satisfaction ;
* taux de conversion ;
* qualité des réponses ;
* temps de résolution.

---

# 6. Support

Règles :

* répondre vite ;
* répondre clairement ;
* tracer chaque demande ;
* escalader les cas sensibles ;
* ne jamais promettre ce qui n'est pas validé ;
* respecter la confidentialité.

---

# 7. Direction

La direction :

* arbitre les cas sensibles ;
* valide les décisions exceptionnelles ;
* suit la stratégie ;
* vérifie les résultats ;
* contrôle les dérives ;
* valide les évolutions documentaires majeures.

---

# 8. Qualité

La qualité doit contrôler :

* la cohérence des annonces ;
* la cohérence des vérifications ;
* la cohérence des réponses ;
* la cohérence des statuts ;
* la cohérence documentaire ;
* la cohérence des services.

---

# 9. Gestion documentaire

Règles :

* toute procédure doit être disponible ;
* tout changement doit être historisé ;
* toute nouvelle version doit être annoncée ;
* tout document obsolète doit être clairement marqué ;
* aucun changement silencieux n'est autorisé.

---

# 10. Gestion des partenaires

Le partenariat doit :

* onboarder ;
* vérifier ;
* suivre ;
* évaluer ;
* suspendre si nécessaire ;
* réactiver si autorisé.

---

# 11. Gestion des utilisateurs

Le support utilisateur doit :

* qualifier les demandes ;
* orienter ;
* rassurer ;
* résoudre ;
* escalader ;
* archiver.

---

# 12. Gestion commerciale

Le commercial doit :

* prospecter ;
* qualifier ;
* présenter ;
* relancer ;
* conclure ;
* activer ;
* fidéliser.

---

# 13. Gestion marketing

Le marketing doit :

* produire des messages cohérents ;
* respecter le brand book ;
* suivre les campagnes ;
* analyser les résultats ;
* coordonner avec le tracking transverse.

---

# 14. Gestion RH

La gestion RH doit :

* suivre les rôles ;
* assurer les formations ;
* documenter les responsabilités ;
* gérer les affectations ;
* garantir la continuité opérationnelle.

---

# 15. Escalade

Escalade requise pour :

* fraude ;
* litige sensible ;
* incident sécurité ;
* incident paiement ;
* conflit documentaire ;
* crise réputationnelle ;
* suspension majeure.

---

# 16. Règles absolues

* ne jamais créer de commission immobilière ;
* ne jamais créer un moteur ;
* ne jamais modifier un workflow technique ;
* ne jamais contourner la validation humaine ;
* ne jamais publier une information non vérifiée lorsqu'elle est critique.

---

# 17. Objectif final

Ce manuel permet de gérer le fonctionnement quotidien de LAWIM de manière claire, disciplinée et traçable.
