# Procédure de vérification des biens

Version 1.0

---

# 1. Objectif

Le présent document définit la procédure de vérification des biens avant publication ou mise en relation.

---

# 2. Périmètre

Cette procédure couvre notamment :

* logement ;
* maison ;
* appartement ;
* terrain ;
* local commercial ;
* immeuble ;
* bien agricole ;
* bien hôtelier ;
* projet immobilier.

---

# 3. Acteurs concernés

* vérificateur ;
* support ;
* partenaire ;
* propriétaire ;
* agent ;
* sécurité ;
* administration.

---

# 4. Niveaux de vérification

* déclaré ;
* pré-vérifié ;
* vérifié ;
* certifié ;
* suspendu ;
* rejeté.

---

# 5. Étapes officielles

1. Réception de la fiche bien.
2. Identification du déclarant.
3. Vérification du lien entre déclarant et bien.
4. Contrôle de la localisation.
5. Contrôle de la disponibilité.
6. Contrôle du prix.
7. Contrôle des photos et médias.
8. Contrôle des documents.
9. Contrôle de cohérence.
10. Détection de fraude et de doublons.
11. Décision.
12. Archivage et reporting.

---

# 6. Checklist minimale

Vérifier au minimum :

* identité du déclarant ;
* droit d'agir sur le bien ;
* adresse ou zone du bien ;
* type de bien ;
* disponibilité réelle ;
* prix cohérent ;
* photos plausibles ;
* documents lisibles ;
* cohérence entre texte, médias et localisation ;
* absence de doublon manifeste.

---

# 7. Documents acceptés

Selon le cas, accepter notamment :

* pièce d'identité ;
* mandat ;
* titre ou document équivalent ;
* preuve d'occupation ou de gestion ;
* autorisation du propriétaire ;
* photos d'origine ;
* plan ou document de localisation ;
* document utile à la démonstration du lien avec le bien.

---

# 8. Éléments suspects

Signaux d'alerte fréquents :

* prix anormalement bas ou élevé ;
* photos déjà utilisées ailleurs ;
* localisation contradictoire ;
* déclarant impossible à joindre ;
* documents flous ou incohérents ;
* plusieurs annonces identiques ;
* pression pour aller hors de LAWIM ;
* demande de paiement non conforme.

---

# 9. Décision

La décision doit être l'une des suivantes :

* accepter ;
* pré-vérifier ;
* demander compléments ;
* suspendre ;
* rejeter ;
* certifier dans les cas les plus solides.

La décision doit toujours être motivée.

---

# 10. Responsabilités

* le vérificateur collecte et analyse ;
* le support relance et centralise ;
* la sécurité traite les cas suspects ;
* l'administration trace la décision ;
* la direction arbitre les cas exceptionnels.

---

# 11. Notifications

Notifier au minimum :

* le déclarant ;
* le partenaire ;
* le responsable interne ;
* la sécurité si suspicion ;
* l'administration si rejet ou suspension.

---

# 12. Reporting

Suivre au minimum :

* nombre de biens vérifiés ;
* nombre de biens suspendus ;
* nombre de biens rejetés ;
* délais moyens de vérification ;
* taux de doublons ;
* motifs de rejet les plus fréquents ;
* répartition par type de bien.

---

# 13. Archivage

Conserver :

* la fiche initiale ;
* les pièces reçues ;
* les captures ou preuves ;
* la décision ;
* la date ;
* le nom du validateur ;
* le motif.

---

# 14. Règles absolues

* ne jamais valider un bien manifestement faux ;
* ne jamais masquer un doute sérieux ;
* ne jamais inventer de preuve ;
* ne jamais créer de commission immobilière ;
* ne jamais publier un bien non vérifié comme certifié.

---

# 15. Objectif final

Cette procédure protège LAWIM, ses utilisateurs et ses partenaires en garantissant des biens crédibles, traçables et correctement classés.
