# Incident Response

STATUT : CIBLE DOCUMENTAIRE ACTIVE

## Purpose

This is the first response path when alerts indicate that backup protection is degraded or broken.

## First actions

1. Identify the affected provider or job.
2. Preserve the latest valid artifacts.
3. Record timestamps and operator identity.
4. Decide whether the system can continue safely.
5. Notify the required stakeholders.

## Communication model

- incident in progress
- backup unavailable
- restore in progress
- service restored
- post-incident report
